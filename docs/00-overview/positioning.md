# Positioning

This repository is a research portfolio for Agent runtime evaluation and reliability engineering.

The point is not to collect generic AI notes. The point is to show judgment about how autonomous agents fail after they leave short demo conditions.

## Core Claim

```text
Agent reliability is a runtime systems problem.
```

Model capability matters, but production failures often appear in a different layer:

- continuity across long sessions
- recovery after interruption
- state consistency
- replayability
- runtime observability
- failure-oriented eval design

## What This Repository Should Demonstrate

- Clear runtime failure taxonomy
- Concrete failure case studies
- Ability to turn incidents into eval fixtures
- Understanding of checkpoint, replay, and observability tradeoffs
- Engineering judgment about real-world agent reliability limits

## What Belongs Here

- Runtime concepts
- Architecture notes
- Failure taxonomy
- Eval methodology
- Case studies
- Research maps

## What Does Not Belong In The Main Path

- Personal career planning
- Daily training logs
- Unstructured scratch notes
- Generic AI industry commentary without a runtime reliability angle

Those materials can live in `docs/archive/` if they are still useful as background.
